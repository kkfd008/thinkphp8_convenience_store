# 便利店零售管理系统（Web版）完整可交付文档

# 文档说明

本文档为便利店零售管理系统（Web版）最终定稿可交付版本，适配PHP8+ThinkPHP6+Layui2.9.10+MySQL/SQLite3双驱动技术栈，适用于社区便利店、小型超市、零食店、文具店等线下零售场景，涵盖完整产品需求、数据库设计、业务流程、部署规范及可执行SQL语句，无冗余内容、无字段冲突，可直接用于项目开发、验收归档与投标汇报。

**版本信息**：V1.0 最终版
**适用环境**：PHP≥8.0、MySQL≥5.7 或 SQLite3、主流浏览器
**核心特性**：MySQL/SQLite3双驱动支持、多终端访问、全业务闭环、多权限管理

# 第一部分 项目整体概述

## 1.1 项目定位

本系统为轻量级B/S架构线下门店管理系统，支持MySQL和SQLite3双数据库驱动，可自由切换，浏览器直接访问即可实现**供货商管理、商品管理、进货管理、库存管理、收银结算、会员管理、数据统计、数据导入导出**全流程业务，支持扫码枪外设，适配电脑、平板、手机、收银机多终端，操作简洁、部署便捷，完美适配小型零售门店日常运营需求。

## 1.2 核心优势

- **双驱动数据库**：支持MySQL（生产环境推荐）和SQLite3（单文件免安装），通过.env配置文件一键切换，兼顾部署便捷性与生产性能

- **全业务闭环**：覆盖供货商-进货-商品-库存-收银-会员-统计全流程，无业务断点

- **多权限分离**：超级管理员、店长、收银员三级角色，权限精准管控，适配多人协同

- **便捷操作**：支持商品Excel批量导入、供货商Excel批量导入、进货单Excel单行/多Sheet复杂格式导入、全业务数据Excel导出

- **实时库存管控**：进货自动增库存、收银自动减库存，库存阈值预警，避免缺货积压

- **跨终端适配**：网页端访问，无需安装客户端，不同设备数据实时同步

## 1.3 运行环境要求

- 服务端：PHP 8.0及以上版本，开启PDO及SQLite3扩展（使用SQLite时），或PDO_MySQL扩展（使用MySQL时）

- 数据库：支持MySQL≥5.7或SQLite3，通过.env中DB_DRIVER配置切换，默认使用SQLite3

- 前端：Chrome、Edge、360极速浏览器等主流现代浏览器

- 外设：支持普通扫码枪、小票打印机（预留对接接口）

- 操作系统：Windows、Linux、macOS全兼容

# 第二部分 完整功能模块需求

## 2.1 权限管理模块

**功能清单**：管理员登录/退出、权限规则管理（树形菜单结构，含图标icon和排序sort字段）、角色管理、管理员账号增删改查、登录会话拦截及权限细粒度校验

**角色权限划分**：

- 超级管理员：拥有系统全部功能权限，可配置角色、管理所有账号与数据

- 店长：拥有业务全权限，可操作商品、进货、库存、会员、统计，无权限配置权限

- 收银员：仅可使用收银台、查看本人收银订单，禁止修改商品、进货、会员数据

## 2.2 供货商管理模块

**功能清单**：供货商新增/编辑/删除、列表查询、状态管理（启用/禁用，通过toggleStatus接口独立切换）、信息维护（名称、联系人、电话、地址、备注）、Excel批量导入、Excel数据导出、导入模板下载

**业务规则**：已关联进货单的供货商禁止删除，仅可禁用；进货单可绑定对应供货商，支持按供货商筛选进货记录。

## 2.3 商品管理模块

**功能清单**：商品增删改查、条码可手工填写且支持按钮点击随机生成13位数字条码、条码唯一性前端+后端双重校验、进货价与零售价分开维护、单位(unit)字段维护、分类管理（独立goods_cate表）、扫码快速查询、Excel批量导入

**业务规则**：商品条码全局唯一，支持手工填写或系统随机生成（genBarcode接口）；关联进货或订单记录的商品禁止删除，且不可修改商品条码；Excel导入自动去重、跳过空行与无效数据，导入时需确保条码唯一性；系统内所有商品关联逻辑均以商品条码（barcode）为准，取消goods_id关联方式；商品需维护最小库存预警（stock_min）和最大库存预警（stock_max），两个字段均支持为空，若为空则不触发对应预警。

## 2.4 进货管理模块

**功能清单**：手动新建进货单、进货单Excel单行导入、多Sheet复杂格式Excel批量导入（自动解析表头/日期/货号，自动创建不存在商品，自动关联供货商）、进货明细录入、供货商绑定、自动核算金额、进货历史查询（按单号/供货商/时间范围筛选）、明细查看、进货数据导出

**核心字段说明**：进货主表记录整单汇总信息（含操作员ID operator_id、备注remark），进货明细表记录单品进货详情，含箱规(box_spec)、箱数(box_count)、散件数量(piece_count)字段，适配整箱+散件混合进货模式；进货明细以商品条码（barcode）关联商品，取消goods_id关联；进货单号格式：JH + Ymd + 三位流水号

**业务规则**：进货单保存后自动累加商品库存（库存增量 = 箱规 × 箱数 + 散件数量），进货单不可删除，永久留存台账；Excel导入自动校验商品条码有效性，无效数据跳过并记录；多Sheet导入支持中文日期解析（xxxx年xx月xx日格式）。

## 2.5 库存管理与预警模块

**功能清单**：库存总览（按商品名称/条码/分类筛选）、库存预警页面（支持按预警类型：低于最小库存/高于最大库存/全部筛选）、预警商品展示（商品名称、当前库存、预警阈值、条码等核心信息）、库存阈值快速更新（updateThreshold接口）、库存数据导出

**业务规则**：商品库存低于stock_min（最小库存预警值）时触发缺货预警（stock_min>0且stock<stock_min），高于stock_max（最大库存预警值）时触发积压预警（stock_max>0且stock>stock_max）；stock_min或stock_max为空的商品，不触发对应类型预警；进货、收银操作实时同步库存数据，同步后自动校验预警状态并更新预警页面展示。

## 2.6 收银台核心模块

**功能清单**：扫码加购（searchGoods按名称/条码搜索）、手动修改数量、购物车编辑、会员搜索与选择（searchMember按姓名/手机号搜索，自动关联会员分类折扣）、支付方式（现金/会员余额）、结算下单（doCheckout）、订单生成

**业务规则**：库存不足商品禁止结算；会员余额不足无法使用余额抵扣；结算后自动扣减库存、扣减会员余额（会员余额支付时），生成不可修改、删除的正式订单；订单号格式：DD + Ymd + 三位流水号

## 2.7 订单管理模块

**功能清单**：订单列表查询（按单号/会员/时间范围筛选）、订单明细查看（关联会员信息）、角色数据隔离（收银员仅看本人订单）、订单数据导出

**业务规则**：订单生成后永久留存，不可修改、删除，作为销售对账核心依据；订单表包含total_amount（商品原价总金额）、discount_amount（折扣金额）、pay_amount（实付金额）、pay_type（支付方式：1现金/2会员余额）等完整金额字段。

## 2.8 会员管理模块

**功能清单**：会员分类管理（member_cate表，支持分类名称和折扣率配置）、会员档案增删改查、会员充值（记录前后余额，可追溯资金变动）、充值记录查询（按会员/时间范围筛选）、会员折扣配置、会员数据导出

**业务规则**：会员手机号全局唯一；会员自动继承所属分类折扣（discount字段）；充值记录留存前后余额（member_recharge表：before_balance/after_balance），可追溯资金变动；有交易记录的会员禁止删除。

## 2.9 数据统计模块

**功能清单**：今日/累计营收统计（pay_amount）、今日/累计订单数统计、近15日销售/进货趋势图（双折线）、商品销量TOP10排行、系统基础数据概况（商品数、会员数、供货商数）

**展示形式**：数字统计卡片+ECharts图表可视化（柱状图展示TOP10、折线图展示趋势），直观呈现门店经营数据。

## 2.10 Excel导入导出规范

### 2.10.1 商品导入模板

商品名称、商品条码、单位、进货价、零售价、库存数量、商品分类、最小库存预警（stock_min）、最大库存预警（stock_max）

### 2.10.2 进货单导入模板（单行模式）

供货商名称、商品条码、商品名称、单位、进货价、零售价、箱规、箱数、散件数量

### 2.10.3 供货商导入模板

名称、联系人、电话、地址、备注、状态（1启用/0禁用）

### 2.10.4 导出范围

支持商品、供货商、进货单、订单、会员、库存全数据导出，导出文件为XLSX格式，时间戳自动转为标准日期格式（Y-m-d H:i:s），金额保留两位小数。

# 第四部分 系统架构说明

## 4.1 技术栈

|层级|技术选型|
|---|---|
|后端框架|ThinkPHP 6|
|数据库|MySQL/SQLite3双驱动（通过.env的DB_DRIVER切换）|
|前端框架|Layui 2.9.10|
|图表库|ECharts|
|Excel操作|PhpSpreadsheet|
|密码加密|MD5|

## 4.2 路由体系

系统采用ThinkPHP路由机制（config/route.php），所有功能通过route/app.php文件定义，总计55条路由，覆盖9大功能模块。URL伪静态后缀为html。

## 4.3 权限控制

- 中间件Auth（app/middleware/Auth.php）负责登录态校验和权限校验
- 超级管理员角色（role_id=1）持有全部auth_rule权限规则ID（逗号分隔）
- 权限校验流程：用户请求 → Auth中间件检查登录态 → 查找auth_rule表中对应规则 → 检查角色rules中是否包含该规则ID
- Index/index和Index/welcome为公共页面，不参与权限校验
- 收银员（role_id=3）在订单模块仅可查看本人经手订单（operator_id过滤）

## 4.4 部署切换说明

通过.env文件配置数据库驱动：

- **SQLite模式**（默认，免安装）：`DB_DRIVER=sqlite`，`DB_DATABASE=database/shop.db`
- **MySQL模式**（生产推荐）：`DB_DRIVER=mysql`，配置`DB_HOST`/`DB_NAME`/`DB_USER`/`DB_PASS`/`DB_PORT`参数
